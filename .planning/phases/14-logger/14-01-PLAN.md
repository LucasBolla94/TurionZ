# Plan 14-01: ActivityLogger Service

## Objective

Create a centralized logging service for the Bolla Network that captures every action
across all components. Every tool call, LLM request, sub-agent operation, and system event
must be logged with structured data.

## Context

- SchemaManager supports on-demand table creation (Caminho C)
- AgentLoop already has iteration logging to console
- SubAgentManager logs to console
- Need: persistent structured logs in PostgreSQL

## Tasks

### Task 1: ActivityLogger core + DB integration

**Files:** `src/infra/ActivityLogger.ts`, `src/types/index.ts`

1. Create `ActivityLogger` singleton class:
   - `log(entry: ActivityLogEntry)` — saves to DB (on-demand table creation via SchemaManager)
   - `logToolCall(agentId, toolName, args, result, duration)` — convenience method
   - `logLlmCall(agentId, model, tokensIn, tokensOut, duration)` — convenience method
   - `logAgentLifecycle(agentId, event, details)` — create/run/complete/fail/cancel
   - `logSystemEvent(component, event, details)` — startup, shutdown, recovery, errors
   - `query(filters: LogQueryFilters)` — query logs with time range, component, agent filters
   - Batch inserts (buffer up to 10 entries, flush every 2 seconds or on buffer full)
   - Graceful flush on shutdown

2. Add `ActivityLogEntry` type to `src/types/index.ts`:
   ```typescript
   interface ActivityLogEntry {
     agentId?: string;
     component: string; // 'agent_loop' | 'sub_agent_manager' | 'controller' | 'gateway' | 'system'
     action: string;
     details: Record<string, unknown>;
     model?: string;
     tokensIn?: number;
     tokensOut?: number;
     durationMs?: number;
     createdAt?: Date;
   }
   ```

3. SQL schema for activity_logs (on-demand):
   ```sql
   CREATE TABLE activity_logs (
     id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
     agent_id UUID,
     component VARCHAR NOT NULL,
     action VARCHAR NOT NULL,
     details JSONB NOT NULL DEFAULT '{}',
     model VARCHAR,
     tokens_in INTEGER,
     tokens_out INTEGER,
     duration_ms INTEGER,
     created_at TIMESTAMP DEFAULT NOW()
   );
   CREATE INDEX idx_activity_logs_agent ON activity_logs(agent_id);
   CREATE INDEX idx_activity_logs_component ON activity_logs(component);
   CREATE INDEX idx_activity_logs_created ON activity_logs(created_at);
   ```

### Task 2: Integration into all components

**Files:** `src/core/AgentLoop.ts`, `src/agents/SubAgentManager.ts`, `src/core/AgentController.ts`, `src/index.ts`

1. **AgentLoop** — After each LLM call, log:
   - `logLlmCall()` with model, tokens, duration
   - After each tool execution: `logToolCall()` with tool name, args summary, result summary, duration
   - On loop start/end: `logAgentLifecycle()`

2. **SubAgentManager** — Log:
   - `logAgentLifecycle()` on create, run, complete, fail, cancel
   - Communication events via `logSystemEvent()`

3. **AgentController** — Log:
   - Message processing start/end with duration
   - Skill routing decisions
   - System initialization

4. **src/index.ts** — Log:
   - Startup event
   - Shutdown event (graceful flush)

## Verification

- Every tool call appears in activity_logs table
- Every LLM call logged with token counts
- Sub-agent lifecycle fully logged
- Batch inserts work (buffer of 10)
- Graceful flush on shutdown
- Query by time range returns correct results
