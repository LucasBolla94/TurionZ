# Plan 15-01: Recovery & Auto-Start

## Objective

Implement full recovery system so TurionZ survives crashes and restarts automatically
with the operating system. On startup: connect DB, verify state, check file integrity,
resume interrupted work, notify user.

## Context

- PostgreSQL has agents table with status tracking
- SchemaManager supports on-demand table creation
- Personality files at .agents/SOUL.md, IDENTITY.md, MEMORY.md
- Gateway handles Telegram connection
- specs/recovery.md has full boot sequence and edge cases

## Tasks

### Task 1: RecoveryManager core

**Files:** `src/infra/RecoveryManager.ts`, `src/types/index.ts`

1. Create `RecoveryManager` singleton:

   - `saveCheckpoint(component, state, iteration?)` — saves to recovery_state table (on-demand)
   - `getCheckpoint(component)` — retrieves last checkpoint
   - `clearCheckpoint(component)` — removes after successful completion
   - `runStartupSequence()` — full boot sequence from spec:
     1. Connect PostgreSQL (3 retries with backoff)
     2. Check recovery_state table for interrupted work
     3. Verify file integrity (SOUL.md, IDENTITY.md, MEMORY.md exist and non-empty)
     4. Check agents table: any with status 'running' or 'waiting'?
     5. Resume or restart interrupted agents
     6. Clean up orphan temp files
     7. Return startup report
   - `verifyCriticalFiles()` — checks personality files exist and are valid
   - `recoverInterruptedAgents()` — finds agents stuck in 'running', decides: resume or restart
   - `getStartupReport()` — formatted message for user notification

2. Safe mode detection:
   - Track crash count in recovery_state
   - If 3 crashes in 10 minutes → safe mode (no sub-agents, no auto-analysis)
   - Safe mode message to user

3. Add checkpoint saving to AgentLoop (before each iteration):
   - Save current iteration number and message count
   - Clear checkpoint on successful completion

### Task 2: Auto-start scripts + boot integration

**Files:** `scripts/autostart-linux.sh`, `scripts/autostart-windows.ps1`, `scripts/autostart-mac.plist`, `src/index.ts`

1. **Linux** (`scripts/autostart-linux.sh`):
   - systemd service file template
   - Install/uninstall commands
   - Auto-restart on failure (RestartSec=5, max 3)

2. **Windows** (`scripts/autostart-windows.ps1`):
   - Task Scheduler XML template
   - PowerShell install/uninstall script
   - Runs on user login

3. **Mac** (`scripts/autostart-mac.plist`):
   - launchd plist file
   - Install to ~/Library/LaunchAgents/
   - KeepAlive + auto-restart

4. **src/index.ts** — integrate RecoveryManager:
   - On startup: run `runStartupSequence()` before initializing components
   - On shutdown (SIGTERM/SIGINT): flush logs, save checkpoints, graceful shutdown
   - Send startup report to user via gateway (if connected)

## Verification

- Checkpoint saved before each AgentLoop iteration
- On simulated crash (kill process), restart recovers checkpoint
- Critical file check detects missing SOUL.md
- Auto-start scripts generated for all 3 OS
- Safe mode triggers after 3 rapid crashes
- Startup report message formatted correctly
