    **Step 2: Create src/skills/ExternalTool.ts.**
    This class extends `BaseTool` and executes cross-language scripts via `child_process.spawn`.

    - Constructor takes: `name: string` (prefixed with skill name, e.g. `"prd-manager.validate_prd"`), `description: string`, `parameters: Record<string, unknown>`, `command: string` (e.g. `"python"`, `"bash"`, `"node"`, `"tsx"`), `scriptPath: string`, `timeoutMs: number = 30000`.
    - `execute(args)` method:
      - Spawns `child_process.spawn(this.command, [this.scriptPath], { timeout: this.timeoutMs })`
      - Writes `JSON.stringify(args)` to stdin, then closes stdin
      - Collects stdout and stderr
      - On exit code 0: try to `JSON.parse(stdout)` — if it has `.success` and `.output` fields, return as `ToolResult`. Otherwise return `this.success(stdout.trim())`.
      - On non-zero exit: return `this.error(stderr || stdout || 'Process exited with code ' + code)`
      - On timeout: return `this.error('Tool execution timed out after ' + this.timeoutMs + 'ms')`
    - Import `spawn` from `child_process`, `BaseTool` from `../tools/BaseTool`, `ToolResult` from `../types`.

    **Step 3: Create src/skills/SkillToolBridge.ts.**
    This class scans a skill's `tools/` directory and creates `ExternalTool` instances.

    - Constructor: no arguments. Set `private pythonCmd` to `process.platform === 'win32' ? 'python' : 'python3'`.
    - Method `loadSkillTools(context: SkillContext): BaseTool[]`:
      - If `!context.toolsDir` return empty array.
      - Read directory with `fs.readdirSync(context.toolsDir)`.
      - For each file, get extension. Map extension to command:
        - `.ts` -> `'tsx'`
        - `.js` -> `'node'`
        - `.py` -> `this.pythonCmd`
        - `.sh` -> `'bash'`
        - Other -> skip (continue)
      - Look up tool metadata from `context.metadata.tools` array. The current `SkillMetadata.tools` is `string[]` (just names). Try to match tool name by filename without extension (e.g., file `validate_prd.py` matches tool name `validate_prd`).
      - If no match found in metadata tools array, use filename as name and generate a generic description.
      - Create `new ExternalTool(prefixedName, description, parameters, command, fullPath)` where:
        - `prefixedName = context.metadata.name + '.' + toolName` (prevents collisions per research Pitfall 3)
        - `description = 'Tool ' + toolName + ' from skill ' + context.metadata.name` (when no manifest)
        - `parameters = { type: 'object', properties: { input: { type: 'string', description: 'Input for the tool' } } }` (generic fallback)
      - Return array of BaseTool instances.
    - Method `unloadSkillTools(skillName: string, registry: ToolRegistry): void`:
      - Get all tool names from `registry.listNames()`
      - Filter names starting with `skillName + '.'`
      - Call `registry.unregister(name)` for each matched name
    - Import: `fs`, `path`, `BaseTool`, `ToolRegistry`, `SkillContext` from `./SkillExecutor`, `ExternalTool` from `./ExternalTool`.

    Class `SkillWatcher`:
    - Private fields: `watcher: chokidar.FSWatcher | null = null`, `debounceTimer: NodeJS.Timeout | null = null`, `debounceMs: number`.
    - Constructor takes `debounceMs: number = 250`.
    - Method `start(skillsDir: string, onReload: () => void): void`:
      - If `this.watcher` is not null, call `this.stop()` first.
      - Create watcher: `chokidar.watch(skillsDir, { depth: 2, ignoreInitial: true, persistent: true })` (depth 2 to catch tools/ subdirectory changes).
      - Listen on `'all'` event. On any event, clear existing debounce timer, set new timer with `this.debounceMs` delay that calls `onReload()`.
      - Log: `[SkillWatcher] Watching ${skillsDir} for changes.`
    - Method `stop(): void`:
      - If `this.watcher`, call `this.watcher.close()`, set to null.
      - If `this.debounceTimer`, clear it.
      - Log: `[SkillWatcher] Stopped watching.`

    - Create new `loadAll(): SkillMetadata[]` that:
