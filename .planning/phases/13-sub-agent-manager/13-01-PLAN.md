# Plan 13-01: SubAgentManager Robustness & Metrics

## Objective

Harden the existing SubAgentManager with all edge cases from specs/sub-agents.md:
circular dependency detection (EC-03), verifier retry loop max 3 (EC-05),
auto-verifier at all levels (EC-06), metrics persistence, and proper progress notifications.

## Context

- SubAgentManager already exists at `src/agents/SubAgentManager.ts` with basic CRUD, run, communicate, cancel
- Missing: circular dep detection, verifier retry limit, metrics save to DB, integration as tool
- Spec: specs/sub-agents.md sections 6.3-6.5, 11

## Tasks

### Task 1: Circular dependency detection + Verifier robustness

**Files:** `src/agents/SubAgentManager.ts`

1. Add `detectCircularDependency(agentId, dependsOnId)` method:
   - Walk the dependency graph from `dependsOnId` upward
   - If we find `agentId` in the chain → throw clear error "Circular dependency detected: A → B → A"
   - Call this in `saveDependency()` before inserting

2. Improve `ensureVerifier()`:
   - Accept `maxRetries = 3` parameter
   - After verifier runs, check verdict (PASS/FAIL)
   - If FAIL and retries < max: sub-agent corrects, verifier re-runs
   - If FAIL after max retries: mark as completed with `result_status: 'completed_with_issues'`
   - Works for ALL sub-agents (level 1 AND level 2), not just level 1 workers

3. Add `saveMetrics(agentId, metrics)`:
   - After `runSubAgent` completes, save to agents table: `metrics` JSONB column
   - Include: `{ tokensIn, tokensOut, duration, toolsCalled, model, iterations }`

4. Block sub-sub-agents (level 2) from creating children:
   - In `createSubAgent()`: if parent level === 2, throw "Sub-sub-agents cannot create children (max 3 levels)"

### Task 2: Progress notifications + Communication improvements

**Files:** `src/agents/SubAgentManager.ts`

1. Add periodic progress tracking:
   - In `runSubAgent()`, before each iteration of the agent loop, emit progress
   - Format: `"Sub-agent {role} ({model}): iteration {n}/{max} — tools used: {list}"`
   - Use existing `onProgress` callback

2. Improve `waitForDependencies()`:
   - Use event-based waiting instead of polling (EventEmitter)
   - When agent completes, emit event that resolves waiting promises
   - Keep polling as fallback with configurable timeout (default 5min from spec)

3. Add `getAgentTree(rootId?)` method:
   - Returns full hierarchy: TurionZ → sub-agents → sub-sub-agents
   - Used by AgentController to show status to user

4. Add `getAgentMetrics(agentId)` method:
   - Returns metrics from DB for a specific agent
   - Includes children metrics aggregated

## Verification

- Circular dependency between A→B→A throws error
- Verifier retries max 3 times before giving up
- Sub-sub-agent (level 2) cannot create children
- 4th sub-sub-agent creation blocked
- Metrics saved to DB after agent completes
- Progress notifications fire during execution
- Agent tree shows correct hierarchy
