# Plan 13-02: Sub-Agent Tools & Controller Integration

## Objective

The LLM (Thor) needs tools to create and manage sub-agents during the ReAct loop.
Without these tools, Thor cannot delegate work. Wire SubAgentManager into the
AgentController so everything flows end-to-end.

## Context

- SubAgentManager exists with create/run/communicate/cancel
- ToolRegistry + BaseTool pattern exists (see src/tools/)
- AgentController handles message pipeline but doesn't know about SubAgentManager
- Thor needs to be able to call tools like `create_sub_agent` during reasoning

## Tasks

### Task 1: Sub-Agent built-in tools

**Files:** `src/tools/SubAgentTools.ts`, `src/tools/ToolFactory.ts`

Create 3 new tools extending BaseTool:

1. **CreateSubAgentTool**
   - Name: `create_sub_agent`
   - Params: `{ briefing: string, model: string, skills?: string[], criteria?: string, dependencies?: string[] }`
   - Calls `SubAgentManager.createSubAgent()` then `runSubAgent()`
   - Returns: `{ agentId, status, result }` when agent completes
   - Validates model exists in supported list

2. **CheckSubAgentTool**
   - Name: `check_sub_agent`
   - Params: `{ agentId?: string }` (optional — if omitted, list all active)
   - If agentId: returns status, progress, result
   - If no agentId: returns list of all active sub-agents with status

3. **CommunicateSubAgentTool**
   - Name: `communicate_sub_agent`
   - Params: `{ fromId: string, toId: string, data: object }`
   - Passes data between sub-agents through TurionZ (centralized — REQ-033)

Register all 3 in ToolFactory so they're available in every AgentLoop.

### Task 2: AgentController integration

**Files:** `src/core/AgentController.ts`

1. In `initialize()`:
   - Create SubAgentManager instance
   - Set progress callback to route notifications to user via `this.router.sendProgress()`

2. In `processMessage()`:
   - After agent loop completes, check if any sub-agents are still running
   - If sub-agents active, append status summary to response

3. In `abortUserLoop()`:
   - Also cancel all active sub-agents for that user

4. Add `getSubAgentStatus()` method:
   - Returns formatted status of all active sub-agents
   - Used by the LLM via check_sub_agent tool

## Verification

- Thor can call `create_sub_agent` during ReAct loop
- Sub-agent runs with own AgentLoop and correct model
- Verifier is auto-created and validates work
- Progress notifications reach user via Telegram
- `check_sub_agent` returns current status
- `communicate_sub_agent` passes data through TurionZ
- Aborting user loop also cancels active sub-agents
