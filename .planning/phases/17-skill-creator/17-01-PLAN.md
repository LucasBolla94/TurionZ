# Plan 17-01: SkillCreator Fixed Sub-Agent

## Objective

Create the SkillCreator as a built-in capability — when Thor or the user needs a new skill,
SkillCreator handles the entire lifecycle: understand need → create files → test → install → confirm.

## Context

- Skill system (Phase 12): SkillLoader, SkillRouter, SkillExecutor, SkillToolBridge, SkillWatcher all working
- Hot-reload via SkillWatcher (chokidar) detects new folders in .agents/skills/
- SubAgentManager (Phase 13): can create and run sub-agents with tools
- ExternalTool supports TS/JS/Python/Bash via spawn
- specs/skill-user.md section 4.4 defines SkillCreator flow

## Tasks

### Task 1: SkillCreator service and tools

**Files:** `src/skills/SkillCreator.ts`, `src/tools/builtin/CreateSkillTool.ts`, `src/types/index.ts`

1. Create `SkillCreator` class:

   - `createSkill(request: SkillCreateRequest)` — full lifecycle:
     1. **Understand:** Parse the request (name, description, what it should do, what tools it needs)
     2. **Plan:** Decide structure — which tools, which languages, which templates
     3. **Generate SKILL.md:** Create frontmatter (name, description, version, author, tools list, languages) + detailed instructions
     4. **Generate tools/** (if needed): Create tool scripts in the best language for each task
     5. **Generate templates/** (if needed): Create any template files the skill uses
     6. **Write to filesystem:** Create folder in `.agents/skills/{skill-name}/`
     7. **Verify:**
        - SKILL.md has valid YAML frontmatter
        - Tool files have correct syntax (attempt to parse/compile)
        - Folder is detected by SkillWatcher (hot-reload)
        - SkillLoader can read the new skill
        - SkillRouter can identify the skill from a test prompt
     8. **Fix:** If verification fails, correct and re-verify (max 3 attempts)
     9. **Report:** Return success with skill details

   - `testSkill(skillName: string)` — runs verification suite on existing skill
   - `listInstalledSkills()` — returns all skills with status

2. Create `CreateSkillTool` (BaseTool):
   - Name: `create_skill`
   - Params: `{ name: string, description: string, purpose: string, tools?: string[], languages?: string[] }`
   - Calls `SkillCreator.createSkill()`
   - Returns creation result with skill details

3. Add `SkillCreateRequest` and `SkillCreateResult` types to types/index.ts

### Task 2: Integration and default skill-creator skill

**Files:** `src/index.ts`, `.agents/skills/skill-creator/SKILL.md`, `src/core/AgentController.ts`

1. **Default skill-creator skill:**
   Create `.agents/skills/skill-creator/SKILL.md`:
   ```markdown
   ---
   name: skill-creator
   description: Creates new skills for TurionZ — handles SKILL.md, tools, templates, testing, and installation
   version: 1.0
   author: BollaNetwork
   ---

   # Skill Creator Instructions

   You are the Skill Creator. When the user or TurionZ needs a new skill:
   1. Ask what the skill should do (if not clear)
   2. Use the `create_skill` tool to create it
   3. Confirm the skill was installed and is ready to use
   4. Explain to the user what the new skill can do
   ```

2. **src/index.ts:**
   - Register CreateSkillTool in tool startup
   - Ensure `.agents/skills/skill-creator/` exists on first run

3. **AgentController:**
   - When SkillCreator creates a skill, the SkillLoader cache is auto-invalidated by SkillWatcher
   - No manual intervention needed — hot-reload handles it
   - Add log entry via ActivityLogger when new skill is created

## Verification

- `create_skill` tool callable by Thor during ReAct loop
- Created skill has valid SKILL.md with frontmatter
- Created skill detected by SkillWatcher (hot-reload)
- SkillLoader reads new skill
- SkillRouter can route to new skill
- skill-creator default skill exists in .agents/skills/
